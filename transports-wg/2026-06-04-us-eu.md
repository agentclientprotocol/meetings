## Agenda

- Introductions and Project Context
  - Alex Hancock: Software engineer at Block, open-sourced Goose agent harness, standardized on ACP client interface
  - Vadim Briliantov: New product lead for ACP team at JetBrains
  - Clifford Ressel: CTO at Nori, uses ACP for remote-hosted agents and TUI clients
  - Artem: Poolside AI

- Architectural Philosophies: Stateless vs. Stateful Connections
  - Discussed challenges of building reliable system on top of V1
  - Stateful connections more effective for real-time updates and bidirectional data flow
  - Transitioning to stateless would require significant rewrite
  - Agreed to avoid delaying V2 proposals to accommodate stateless shift

- Current Implementations and SDK Status
  - MVP for proposed transport exists in Goose codebase, submitted as PR to Rust SDK
  - TypeScript client at Atlassian in progress
  - TypeScript client by Poolside AI has demonstrated remote agent communication with Goose

- Finalizing the Specification and Versioning
  - Transport is an additive feature, not a breaking change
  - Consensus to include transport in V1
  - Robustness features (message IDs, reconnection logic) may require V2 updates
  - Transport can be part of V1 with clear caveats guiding users toward V2 for reliability

- Iterative Development and Unstable Labeling
  - "Preview" status should imply stable, tested implementation, possibly behind feature flags
  - "Unstable" labels for SDK-level implementations allow iteration without premature commitment
  - Separate versioning rather than feature flags where possible to signal implementation state

- Implementation Collaboration and Testing
  - Plans to test across various clients including Zed (custom async executor)
  - Ongoing efforts by Atlassian
  - Ben to review pending merge requests and manage versioning for new crate

- International Team Coordination
  - JetBrains working on cloud-based agents using ACP with TypeScript wrappers and Kotlin/Java SDK
  - Plans to sync with JetBrains websocket developers to compare approaches
  - Follow-up call with Australian team for TypeScript coordination

## Decisions

- Remote transport RFD included in V1 as an additive feature
- Versioning strategy: separate versioning scheme for unstable implementation rather than compiler feature flags
